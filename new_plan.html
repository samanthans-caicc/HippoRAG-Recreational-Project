Paper 1: HippoRAG (2024)

Core Idea

HippoRAG is a biologically inspired system that combines Large Language Models (LLMs), Knowledge Graphs (KGs), and the Personalized PageRank (PPR) algorithm to mimic how the neocortex and hippocampus interact in human memory.

Key Contributions

  • Hybrid memory system (LLM + Knowledge Graph)
  • Two-stage pipeline: Offline (KG construction) + Online (retrieval)
  • Single-step multi-hop reasoning using graph traversal (PPR)

Results

  • ~20% improvement over traditional RAG methods
  • Comparable or better performance using single-step retrieval
  • More computationally efficient than iterative retrieval approaches

Pipeline

Documents → Knowledge Graph → Query → PPR Traversal → Answer

System Pipeline

This diagram shows the overall HippoRAG pipeline. Raw documents are first processed by an LLM to construct a Knowledge Graph (KG), which stores relationships between entities. When a query is made, the system performs a Personalized PageRank (PPR) traversal over the graph to identify the most relevant connected information. The retrieved context is then passed to the LLM to generate a final answer. This replaces traditional document retrieval with structured memory traversal.

Detailed Notes & Explanation

Introduction

HippoRAG orchestrates LLMs, knowledge graphs, and the Personalized PageRank (PPR) algorithm to simulate the roles of the neocortex (processing) and hippocampus (memory indexing).

It is evaluated against existing Retrieval-Augmented Generation (RAG) systems, particularly on multi-hop question answering tasks.

Background

Retrieval Augmented Generation (RAG)

RAG allows new information to be introduced into a model without retraining by retrieving relevant documents at query time. However, traditional RAG treats each piece of information independently, making it difficult to perform complex reasoning that requires connections between multiple data points.

Open Information Extraction

Open Information Extraction (OpenIE) is a technique used to extract structured relationships from unstructured text. It typically outputs triples in the form: 

(subject, relation, object)

These triples can then be used to construct a knowledge graph representing relationships between entities.

HippoRAG Concept

HippoRAG introduces associative memory by connecting pieces of information within a knowledge graph. This allows related concepts to be discovered through graph traversal rather than manual comparison.

Instead of searching isolated documents, the system navigates relationships between entities. The LLM is used to generate the knowledge graph, which serves as a memory structure.

Query concepts act like “seeds” that are placed into the graph. The Personalized PageRank (PPR) algorithm then spreads influence through connected nodes, similar to how ideas expand through associative memory in humans.

Knowledge Graph Traversal (PPR)

This diagram illustrates how Personalized PageRank (PPR) operates over a knowledge graph. The query is first mapped to one or more nodes in the graph. From these starting points, probability spreads across connected nodes. Nodes that are more strongly connected to the query receive higher scores. The most relevant node (highlighted as the answer) is selected based on this ranking. This process allows the system to perform multi-hop reasoning in a single step.

Hippocampal Memory Indexing Theory (Biological Motivation)

This theory suggests that long-term human memory relies on two key mechanisms:

  • Pattern Separation: Ensures different experiences are stored as distinct representations.
  • Pattern Completion: Allows full memories to be retrieved from partial inputs.

During encoding, the neocortex processes sensory input into high-level features, which are then passed to the hippocampus for indexing. The hippocampus stores relationships and salient features that help determine how memories are connected.

New information can be integrated dynamically by updating these memory indices, allowing for flexible and evolving knowledge representation.

Implementation

Offline Indexing (Memory Encoding)

  • Use a tuned LLM to extract knowledge graph triples via OpenIE
  • Build a schemaless knowledge graph
  • Use dense retrieval encoders to improve connectivity between nodes

Online Retrieval

  • LLM extracts key query entities (salient concepts)
  • Entities are matched to nodes in the knowledge graph
  • Matched nodes become “query nodes” (starting points)
  • PPR algorithm traverses the graph from these nodes
  • Relevant nodes and passages are ranked and retrieved

This process allows the system to move from partial information to complete answers, mirroring how human memory reconstructs knowledge from incomplete cues.

Paper 2: HippoRAG 2 (2025)

Core Idea

HippoRAG 2 extends the original HippoRAG framework into a more complete memory system, aiming to support factual recall, associative reasoning, and sense-making simultaneously.

Key Improvements

  • Integration of passage-level information into the knowledge graph
  • Improved query understanding through query-to-triple matching
  • LLM-assisted filtering to improve retrieval accuracy

Results

  • Improved performance across factual, associative, and reasoning tasks
  • ~7% improvement in associative memory benchmarks
  • Better balance between retrieval accuracy and reasoning capability

Detailed Notes & Explanation

Introduction

HippoRAG 2 builds upon the first version by addressing a key limitation in existing structure-augmented RAG systems such as HippoRAG v1, GraphRAG, RAPTOR, and LightRAG.

While these systems improve reasoning and associativity, they often perform worse than standard dense retrieval on simple factual question answering tasks. This reveals a critical gap: a true memory system must perform well across all types of memory.

The goal of HippoRAG 2 is to unify three types of memory:

  • Factual Memory (direct retrieval)
  • Associative Memory (connections between ideas)
  • Sense-Making (multi-hop reasoning)

To achieve this, the system retains the PPR-over-KG backbone but enhances it by integrating richer contextual information and using the LLM more actively during retrieval.

Key Improvements Over v1

1. Dense–Sparse Integration (Passage Nodes)

HippoRAG 2 introduces passage nodes into the knowledge graph. These nodes are connected to extracted phrases using "contains" relationships.

This allows the graph to represent both:

  • Concept-level knowledge (sparse, entity-based)
  • Contextual information (dense, passage-based)

This significantly improves factual recall while maintaining reasoning capabilities.

2. Deeper Contextualization (Query-to-Triple Linking)

Instead of mapping individual entities (NER → node), the system embeds the entire query and matches it directly against triples in the knowledge graph.

Triples preserve relational context, which individual entities often lose. This leads to better alignment between the query and stored knowledge.

Empirical results show that this approach improves retrieval performance significantly (over 10% improvement in ranking metrics such as R@5).

3. Recognition Memory (LLM-Based Filtering)

After retrieving candidate triples, the LLM is used to filter out irrelevant ones before they are used to seed the PPR algorithm.

This mimics the human memory process of:

  • Recall: retrieving possible candidates
  • Recognition: selecting the correct ones

This step improves precision and reduces noise in graph traversal.

HippoRAG 2 Retrieval Pipeline

This diagram highlights the improvements introduced in HippoRAG 2. Instead of directly mapping queries to entities, the system matches the entire query against triples in the knowledge graph, preserving relational context. The LLM then filters these candidate triples to remove irrelevant ones, mimicking human recognition memory. The refined set of nodes is used as input to the PPR algorithm, resulting in more accurate and context-aware retrieval.

Updated Retrieval Pipeline

  • Query is embedded and matched to graph triples
  • Top candidate triples are retrieved
  • LLM filters relevant triples (recognition step)
  • Filtered nodes are used as seeds for PPR traversal
  • Graph traversal ranks relevant nodes and passages
  • Final context is passed to the LLM for answer generation

Key Insight

HippoRAG 2 shifts the system from a retrieval-based architecture to a true memory system. Instead of simply fetching relevant documents, it dynamically reconstructs knowledge by combining structured relationships, contextual passages, and LLM reasoning.

Section 3: GitHub Repository (Implementation)

Core Idea

The HippoRAG GitHub repository provides a full implementation of the system described in the papers, including knowledge graph construction, graph-based retrieval using Personalized PageRank (PPR), and evaluation pipelines.

What the Repository Contains

  • Knowledge Graph construction pipeline using LLM-based triple extraction
  • Graph traversal engine implementing Personalized PageRank (PPR)
  • Query processing and entity/triple matching components
  • Evaluation scripts for benchmarking multi-hop QA tasks

Detailed Explanation

1. Knowledge Graph Construction

The repository builds a schemaless knowledge graph using Open Information Extraction (OpenIE). An LLM is used to extract triples of the form: 

(subject, relation, object)

These triples form nodes and edges in the graph, representing relationships between concepts. Dense retrieval encoders are used to improve connectivity by linking semantically similar nodes.

2. Retrieval Engine (PPR)

The core retrieval mechanism is Personalized PageRank (PPR), which enables efficient multi-hop reasoning over the graph.

  • Query is mapped to graph nodes (or triples)
  • These nodes act as seeds
  • PPR spreads probability across connected nodes
  • Highly relevant nodes receive higher scores

This replaces traditional iterative retrieval with a single graph traversal step.

3. Query Processing

The system extracts important concepts from the query and links them to the graph. In newer versions (HippoRAG 2), this is done using query-to-triple matching rather than simple entity linking, preserving relational context.

4. LLM Integration

The LLM is used in multiple stages:

  • Triple extraction (offline memory encoding)
  • Query understanding (entity or triple extraction)
  • Filtering relevant information (recognition step in v2)
  • Final answer generation

5. End-to-End Pipeline

Documents → LLM → Knowledge Graph
                    ↓
Query → Matching → Graph Nodes
                    ↓
              PPR Traversal
                    ↓
          Ranked Context Retrieval
                    ↓
               LLM Answer

How to Use the Repository

cd HippoRAG

Step 2: Install Dependencies

pip install -r requirements.txt

Step 3: Build Knowledge Graph

python build_kg.py

Step 4: Run Retrieval

python run_retrieval.py

Step 5: Evaluate

python evaluate.py

Key Insights

  • The repository shows that memory can be externalized into a graph structure
  • PPR enables efficient multi-hop reasoning without iterative retrieval
  • System performance depends heavily on graph quality and connectivity

Challenges

  • Knowledge graph construction is computationally expensive
  • Performance depends on quality of extracted triples
  • Entity/triple matching must be accurate

Section 4: Reproducing HippoRAG 2

Core Objective

The goal of reproducing HippoRAG 2 is to build a non-parametric memory system that enables an LLM to perform factual retrieval, associative reasoning, and multi-hop inference without retraining.

Detailed Methodology

1. Data Collection

Begin by collecting a corpus of documents. The quality and diversity of this data are critical, as they directly impact the quality of the knowledge graph.

2. Knowledge Graph Construction

Triple Extraction

Use an LLM with Open Information Extraction (OpenIE) to extract triples: 

(subject, relation, object)

Graph Creation

Construct a schemaless knowledge graph where:

  • Nodes represent entities or concepts
  • Edges represent relationships

Connectivity Enhancement

Use dense retrieval embeddings to connect semantically similar nodes, improving graph traversal performance.

3. Query Processing

Query Representation

Instead of extracting only named entities, encode the full query to preserve context.

Query-to-Triple Matching

Match the query against triples in the knowledge graph to identify relevant starting points.

4. Recognition Step (LLM Filtering)

Use the LLM to filter candidate triples before traversal. This step removes irrelevant nodes and improves precision, mimicking human recognition memory.

5. Graph Traversal (PPR)

Apply Personalized PageRank using the filtered nodes as seeds.

  • Probability spreads through the graph
  • Relevant nodes accumulate higher scores
  • Multi-hop reasoning emerges naturally

6. Context Retrieval

Select top-ranked nodes and retrieve their associated passages to form the context for answer generation.

7. Answer Generation

Feed the retrieved context into the LLM to generate the final response.

Full Pipeline Overview

Corpus → Triple Extraction → Knowledge Graph
                    ↓
Query → Triple Matching → Candidate Nodes
                    ↓
          LLM Filtering (Recognition)
                    ↓
            PPR Graph Traversal
                    ↓
           Ranked Context Selection
                    ↓
               LLM Answer

Key Insights

  • Performance depends heavily on knowledge graph quality
  • PPR enables efficient multi-hop reasoning without iterative retrieval
  • LLM plays a dual role: memory encoding and retrieval refinement

Challenges

  • High computational cost during graph construction
  • Requires accurate triple extraction
  • Query-to-triple matching must be precise

Final Insight

Reproducing HippoRAG 2 is less about scaling the model and more about building a high-quality, well-connected memory structure that the model can efficiently traverse.
