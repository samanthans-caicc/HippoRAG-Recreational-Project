Paper 1

Biological Inspiration: Implement Hippocampal Memory into an LLM


Introduction

HippoRAG orchestrates LLMs, knowledge graphs, and the Personalized PageRank (PPR) algorithm to mimic the different roles of the neocortex and hippocampus in human memory.

It compares HippoRAG with existing RAG models on multi-hop question answering tasks.


Results

  • HippoRAG is approximately 20% better.
  • Achieves comparable or better results with single-step retrieval.
  • More cost-efficient compared to traditional approaches.

Background

Retrieval Augmented Generation (RAG)

RAG allows new information to be presented to a model without retraining. However, traditional RAG has limitations: each item is treated independently, making complex reasoning difficult without explicit connections.

Open Information Extraction

A method used to extract structured information (such as triples) from unstructured text.


HippoRAG Concept

HippoRAG introduces associative memory to connect ideas, making it easier to retrieve related information.

When data is interconnected, it becomes possible to traverse relationships efficiently instead of manually comparing isolated data points, as required in traditional RAG.

The design uses an LLM as a corpus to construct Knowledge Graphs (KG), which act as hippocampal memory. A Personalized PageRank (PPR) algorithm is then applied to traverse the KG using query concepts as seeds.

These query concepts act like seeds: they are planted based on key data and expand through the graph, with PPR guiding their growth and traversal.

Hippocampal Memory Indexing Theory (Biological Motivation)

This theory proposes that long-term human memory consists of components that achieve two objectives:

  • Pattern Separation: Ensures distinct experiences are uniquely represented.
  • Pattern Completion: Enables full memory retrieval from partial inputs.

Pattern separation occurs during memory encoding. The neocortex processes stimuli into high-level features, which are then routed through parahippocampal regions to the hippocampus for indexing.


Indexed memories include salient information that helps determine relationships between data points. New information can be integrated dynamically by updating hippocampal indexing.

Implementation:

Offline Indexing (Memory Encoding)

  • Uses a tuned LLM to extract knowledge graph triples.
  • Knowledge graph is schemaless using open information extraction.
  • Dense retrieval encoders improve connectivity between nodes.

Online Retrieval

  • The LLM extracts key query entities (salient concepts).
  • Entities are matched to nodes in the knowledge graph.
  • Matched nodes become starting points (query nodes).
  • PPR algorithm traverses the graph from these nodes.
  • Relevant information is ranked and retrieved.

Paper 2

Introduction:

In expansion to Version 1 of the HippoRAG paper, we present further advancements and improvements. The core problem of this work Structure-augmented RAG methods (HippoRAG v1, GraphRAG, RAPTOR, LightRAG) improve on associativity or sense-making but underperform plain dense retrieval on simple factual QA. A true long-term memory system needs to be strong across all three memory types: factual, sense-making, and associative. The key idea is to keep HippoRAG's PPR-over-KG backbone, but fix its concept-only bias by integrating passages more deeply into the graph and using the LLM more actively at retrieval time.


Key Improvements over v1:

Dense–sparse integration (passage nodes): Add passage nodes to the KG, connected to their extracted phrases via "contains" edges. This lets the graph carry both concept-level (sparse) and context-level (dense) information instead of relying only on phrase nodes.
Deeper contextualization (query-to-triple linking): Instead of v1's NER-to-node linking, embed the entire query and match it against whole triples in the KG. Triples carry relational context that bare entities lose. Ablations show query-to-triple beats NER-to-node by ~12.5% R@5 on average.
Recognition memory (LLM triple filtering): After retrieving top-k candidate triples, the LLM filters them to keep only the relevant ones before they seed PPR — modeling the recall/recognition split in human memory.
